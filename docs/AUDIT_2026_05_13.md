## 1 — TL;DR exécutif

**Note globale : 7.0 / 10.**

Dog Universe est une codebase **mature pour son stade** (≈ 745 tests Vitest verts, instrumentation Sentry + Guardian + Heartbeat, feature flags DB-backed, soft-delete cohérent, idempotence multicouche). Les fondations sont saines, le projet est en production stable et survit à des features ambitieuses (Pet Taxi GPS, AI Guardian, restore additif).

Les **3 axes prioritaires** pour passer de "fonctionne en prod" à "scalable industrialisé" :

1. **🔴 Activer le PgBouncer Supabase** (port 6543) — déjà documenté en TODO depuis des semaines. Sans pooler, le mur des ~500 req/s arrive dès qu'un cron + 5 réservations concurrentes tombent en même temps.
2. **🟠 Couverture API** — 14 % des routes ont des tests directs (18 / ~131). Pour les routes de mutation (POST/PATCH/DELETE) c'est la dette la plus risquée. Pas de garde-fou anti-régression sur des chemins financiers.
3. **🟠 Adoption uniforme des wrappers** — `withSchema`, `withSpan`, `requireRole`, `notDeleted()` existent mais sont sous-utilisés (4–10 % des sites). Le code récent les utilise, le code legacy non. Risque de drift silencieux.

Aucun finding **critique exploitable immédiatement** ne ressort. Les claims spéculatifs des agents sécurité ont été triés (cf §4).

### 2.2 Faiblesses

#### 🟠 HIGH

- **A-H1. Service layer non généralisé.** Seuls `booking-*` sont extraits en services. Invoices, products, pets, loyalty restent dans les routes (god-files). Conséquence : duplication de logique métier entre `/api/admin/invoices` et `/api/invoices`, testabilité limitée. **Fix** : extraire `invoices.service.ts` + `loyalty.service.ts` (M, ~2 semaines pour les deux). [`src/app/api/invoices/route.ts`, `src/app/api/admin/products/route.ts`]

- **A-H2. `withSchema` adoption ~5 %.** ~6 routes l'utilisent, ~125 font de la validation Zod ad-hoc inline. Conséquence : style hétérogène, error envelopes inconsistantes. **Fix** : convention "toute nouvelle route POST/PATCH passe par `withSchema`" + migration progressive (L). [`src/lib/with-schema.ts`]

- **A-H3. `withSpan` adoption nulle.** Helper observability exporté mais zero call-site grep. Conséquence : pas de traces Sentry sur les opérations longues (pricing recalc, taxi geofencing, restore additif). **Fix** : wrapper les paths critiques (S, ~2h pour 5 spots). [`src/lib/observability.ts`]

#### 🟡 MEDIUM

- **A-M1. God-files UI restants.** `NewBookingForm.tsx` 748L, `ProductsClient.tsx` 702L, `BackupsClient.tsx` 640L (refondue ce jour mais reste épaisse), reservation `[id]/page.tsx` ~680L. Charge cognitive haute, régressions visuelles fréquentes. **Fix** : extraction sous-composants par section (M par fichier).

- **A-M2. Imports relatifs profonds.** `'../../../../../auth'` dans 15+ pages. Cosmétique mais fragile au refactor. **Fix** : ajouter alias `@auth` dans `tsconfig.json:paths` puis sed (S).

- **A-M3. Pagination hétérogène.** Bookings + invoices utilisent cursor-based (`src/lib/pagination.ts`), products fait offset/limit. UX inconsistante. **Fix** : uniformiser sur cursor (M).

- **A-M4. `notDeleted()` helper non adopté.** ~99 occurrences inline `deletedAt: null` malgré l'existence du helper. Risque : oublier le filtre dans une nouvelle route. **Fix** : codemod + lint rule custom (M).

#### 🟢 LOW

- **A-L1. Multi-tenant scaffold confirmé propre.** PR #21 a retiré `Tenant` ; grep `tenantId|multi-tenant` → 0 hit. Pas de dette résiduelle. ✅

- **A-L2. Commentaires sur soft-delete Edge.** `src/lib/prisma.ts` mentionne encore "future $extends wrapper". Documenter que l'approche manuelle est intentionnelle, pas un TODO. (S)

- **P-L1. Supabase Read Replica.** Aucune doc indique un read replica en place. Si la pression analytique grossit (`/admin/analytics`, MV refresh), un replica dédié aux longues lectures soulagerait l'instance primary. (XL)

Bien câblé sur les hot paths : capacity (5 min), loyalty grade (5 min), notif unread count (30 s), admin counts via `unstable_cache` tag (30 s), availability calendar (5 min). ## 4 — Sécurité

### 4.1 Tri des findings de l'agent offensif

L'agent security a remonté 18 items. Après vérification de chaque "scenario" :

| ID Agent | Statut après audit humain | Sévérité retenue |

| #1 Privilege escalation JWT role | ⚠️ Partiellement valide — le role est refetch via `dbUser` à chaque renew (`auth.ts:79-118`). Aucun endpoint `PATCH /api/profile/role` n'existe. Reste : ADMIN qui change son propre password n'incrémente pas le tokenVersion des SUPERADMINs (ce qui est normal). **Reclassifié 🟢 LOW.** | LOW |

| #3 SSRF via Nominatim | ⚠️ Pas de SSRF réel (Nominatim n'accepte pas d'URL arbitraire en input — l'input est lat/lng). Reste : abuse possible des quotas Nominatim. Bucket existant `geocode: 30 / 60 min`. **Reclassifié 🟢 LOW** (cosmétique). | LOW |
| #4 Rate-limit fail-closed masquant attaque | ✅ Vérifié `src/middleware/rate-limit.ts:349`. Le code fail-closed quand Upstash KO. Trade-off accepté. **🟡 MEDIUM** (ajouter LRU in-memory fallback). | MEDIUM |
| #5 CSV injection export factures | ⚠️ À vérifier. CLAUDE.md dit qu'un `escapeCsv()` a été ajouté (2026-03-20). À confirmer en lisant `/api/admin/invoices/export/route.ts`. | À vérifier |
| #6 NextAuth JWT refresh window | ⚠️ Demi-vrai. `maxAge: 24h` est long. Mais le `jwt()` callback re-fetch `tokenVersion` à chaque renouvellement, donc l'invalidation arrive au plus tard à la prochaine action authentifiée. **🟢 LOW** (réduire à 12h cosmétique). | LOW |
| #7 HMAC Sentry — order length vs hex | ✅ Vrai. Réordonner length avant regex. **🟡 MEDIUM** (defense-in-depth, pas exploit réel). | MEDIUM |

### 4.2 Findings retenus après tri

#### 🟠 HIGH (nouveaux ou à confirmer)

- **S-H1. Vérifier que l'API est bloquée pendant `totpPending=true`** ⚠️. Si l'API accepte les calls pendant le 2FA pending, c'est un bypass complet du second facteur sur les routes JSON. **Action** : grep `totpPending` dans `src/app/api/**` et confirmer le middleware ou le guard. (S, 30 min). [`src/middleware.ts`]

- **S-H2. Vérifier `escapeCsv()` sur tous les exports** ⚠️. CLAUDE.md affirme que c'est en place mais le code n'a pas été relu dans cette session. Routes à check : `/api/admin/invoices/export`, `/api/admin/clients/export` (s'il existe), `/api/admin/loyalty/export`. (S, 15 min).

#### 🟡 MEDIUM

- **S-M1. TOTP token hashé.** Migrer `lastTotpToken` de plaintext à `sha256(token)`. Ajout d'une migration SQL + update `src/lib/totp.ts`. (M, ~2h). [`src/lib/totp.ts`]

- **S-M2. Rate-limit LRU fallback.** Si Upstash latency spike → tout est bloqué. Ajouter un cache in-memory module-scope (Node LRU 1000 entries × 60 s) qui prend le relais ≤ 30 s. (M, ~3h). [`src/middleware/rate-limit.ts`]

- **S-M3. HMAC order fix.** Inverser length-check vs hex-regex. (S, 5 min). [`src/app/api/webhooks/sentry/route.ts:62-81`]

- **S-M4. NextAuth JWT `maxAge` = 12h.** Réduit la fenêtre d'utilisation post-révocation. (S, 1 ligne). [`auth.ts:15`]

#### 🟢 LOW / INFO

- **S-L1.** `notes` Zod : refuser `\n\n` doublé pour défense-en-profondeur SMS (`z.string().max(2000).regex(/^[^\r\n]*(\n[^\r\n]*)*$/)`).
- **S-L2.** next-auth toujours en beta (`5.0.0-beta.30`). Surveiller GA et upgrader.
- **S-L3.** bcrypt 12 rounds : OK pour 2026. Réévaluer en 2028.

## 11 — Conclusion

Dog Universe est une **codebase production-ready** pour son stade actuel (PME marocaine). Les choix techniques sont défendables (Next 15 App Router + Prisma + Supabase = stack moderne), les patterns réutilisables existent (`defineCron`, `withSchema`, `withSpan`, `requireRole`, `notDeleted`), l'observabilité est déjà au-dessus de la moyenne (Sentry + Guardian + Heartbeat + page `/status` publique).

Les 3 points de levier les plus rentables :

1. **Une action manuelle Supabase** (PgBouncer) débloque la scale serverless
2. **2 sprints de tests API** rendent les chemins financiers défendables face à la régression
3. **Un sprint d'adoption uniforme** des wrappers existants élimine la moitié de la dette structurelle

Les manques produit (RBAC, paiement en ligne, WhatsApp 2-way) sont des **choix business**, pas de la dette technique — chacun débloque un palier de croissance différent.
